**Step 4: 钉钉 API 客户端 + Token 管理器** ✅
- `apps/dingtalk/exceptions.py` — DingTalkError / TokenRefreshError / APIRequestError + check_response() 统一错误处理
- `apps/dingtalk/token_manager.py` — TokenManager 类（模块级单例 token_manager）：
  - 内存缓存 Access Token，TTL 7000s（7200-200 安全余量）
  - 线程安全（threading.RLock），自动过期刷新

- `apps/dingtalk/client.py` — DingTalkClient（模块级单例 client）：
  - `get_report_list(start, end, cursor, size)` → POST /topapi/report/list
  - `get_user_info_by_code(auth_code)` → POST /topapi/v2/user/getuserinfo
  - 所有方法在 Demo 模式返回中文 mock 数据（5 个虚拟用户、7 个模板字段、模拟分页）
  - 自动注入 access_token，统一异常处理
- `apps/dingtalk/models.py` — SyncLog 模型：
  - sync_type: incremental / deep / full / manual
  - status: running / success / failed / partial
  - 记录 records_fetched / parsed / skipped + request/response JSON
- `apps/dingtalk/admin.py` — SyncLog 注册到 Admin，状态彩色标签 + 耗时展示
