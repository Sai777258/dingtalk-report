### 2026-07-02 — 数据清理 + 导入命令增强

**清理旧测试日志** ✅
- 问题：外部 MySQL `daily_logs` 表中有旧测试数据（ID 3, 9, 12, 17, 26），之前已导入本地 SQLite，但源库清理后本地仍保留

- 根因：导入命令只做 INSERT + 去重跳过，从不 DELETE 或 UPDATE 已导入数据
- 清理：通过 Django shell 删除 5 条 `ext_daily_*` WorkReport，CASCADE 自动清理关联的 ReportContent (5条) 和 WorkEntry (16条)，共删除 26 条记录
- 清理后：104 → 99 WorkReport，278 → 262 WorkEntry，ext_daily_ 报告：56 → 51

**`import_from_external_db` 新增 `--clean-stale` 选项** ✅
- 新增 `_clean_stale_records(conn, dry_run)` 方法：
  - 查询外部 MySQL `daily_logs` 表中当前存在的 ID
  - 对比本地所有 `ext_daily_*` 记录，找出源库已删除的"过时记录"
  - 用 WorkReport.delete() 级联清理（ReportContent + WorkEntry）
- 三处调用点：
  - `--dry-run` 模式：仅列出待删除记录，不实际删除
  - 无新数据时：仍可执行清理（`--clean-stale` 独立于导入逻辑）
  - 正常导入后：关闭导入连接 → 重新打开连接 → 清理过时 → 写入 SyncLog
- SyncLog 增强：response_summary 新增 `stale_checked` / `stale_deleted` / `stale_to_delete` 字段
- 终态输出：清理完成后打印过时记录数

**数据导入架构现状总结** ✅
