# 2026-07-03 工作日志

## 任务：排查 + 修复工作日志管理系统自动跳转登录页问题

### 问题现象
Demo 测试时，用户登录后约 5 分钟会自动跳转回登录界面。

### 根因
JWT Access Token 使用 SimpleJWT 默认有效期（5分钟），且前端未实现 Token 刷新机制。
401 拦截器收到 401 后直接 logout() + 跳转，未尝试用 refresh token 刷新。

### 修复方案：实施方案B — 前端 Refresh Token 刷新机制

#### 修改的文件

1. **`apps/accounts/urls.py`**（后端）
   - 添加 `TokenRefreshView` 路由：`POST /api/auth/token/refresh/`

2. **`frontend/src/api/auth.js`**（前端）
   - 新增 `refreshToken(refresh)` 函数
   - 使用独立干净 axios 实例（无拦截器），避免刷新时无限循环

3. **`frontend/src/stores/auth.js`**（前端）
   - 新增 `setTokens(access, refresh)` action
   - `fetchUser()` 去掉直接 `logout()` 调用，改为仅清除 `user.value`

4. **`frontend/src/api/index.js`**（前端 — 核心）
   - 重写 401 响应拦截器：尝试刷新 token → 成功则重试原请求 → 失败则 logout
   - 并发请求队列机制，防止多个 401 同时触发多次刷新

#### 验证结果
- 后端 URL 注册 ✅
- 后端冒烟测试 ✅（有效 token 返回 200，无效返回 401）
- 后端单元测试 29/29 通过 ✅
- 前端 Vite 构建成功 ✅

- 配置 `ROTATE_REFRESH_TOKEN: True` 提升安全性
